springMVC+Mybatis框架搭建实例

SpringMvc4+Mybatis3框架搭建实例

前景摘要

为什么要使用springMVC？springMVC和struts2有什么区别？

spring Web MVC是一种基于Java的实现了Web MVC设计模式的请求驱动类型的轻量级Web框架，即使用了MVC架构模式的思想，将web层进行职责解耦，基于请求驱动指的就是使用请求-响应模型，框架的目的就是帮助我们简化开发

1.springmvc基于方法开发的，struts2基于类开发的。springmvc将url和controller里的方法映射。映射成功后springmvc生成一个Handle对象，对象中只包括了一个method。方法执行结束，形参数据销毁。springmvc的controller开发类似web service开发。

2.springmvc可以进行单例开发，并且建议使用单例开发，struts通过类的成员变量接受参数，无法使用单例，是能使用多例

3.通过实际测试，struts2速度慢，在于使用struts标签，如果使用struts建议使用jstl

MyBatis是什么

MyBatis的前身叫iBatis，本是apache的一个开源项目，2010年这个项目由apache software迁移到了google code，并且改名为MyBatis。MyBatis是支持普通SQL查询，存储过程和高级映射的优秀持久层框架。

MyBatis消除了几乎所有的JDBC代码和参数的手工设置以及结果集的检索。MyBatis使用简单的XML或注解用于配置和原始映射，将接口和Java的POJOs（Plan Old Java Objects，普通的Java对象）映射成数据库中的记录。

def __getVideoFileUrl (self, videoInfo) :
    videoInfo = json.JSONDecoder().decode(videoInfo)
    if 'security' in videoInfo['data']:
        ep = videoInfo['data']['security']['encrypt_string']
        print("ep:"+ep)
    else :
        ep = False

    if ep :
        oip   = videoInfo['data']['security']['ip']
        vid   = videoInfo['data']['video']['encodeid']
        temp  = self.__yk_e('becaf9be', base64.decodestring(ep))
        sid   = temp.split('_')[0]
        token = temp.split('_')[1]
        ep    = urllib.quote(base64.encodestring(self.__yk_e('bf7e5f01', str(sid) + '_' + str(vid) + '_' + str(token))), '')[0:-3]
        fileUrl = self.fileUrlPrefix + '&ep=' + str(ep) + '&oip=' + str(oip) + '&sid=' + str(sid) + '&token=' + str(token) + '&vid=' + str(vid) + '&type=' + self.videoTypeList[self.videoType]+'&special=true'
    else :
        fileUrl = False

    print(fileUrl)

    return fileUrl

def __yk_e (self, a, c) :
    f = 0
    i = 0
    h = 0
    b = {}
    e = ''
    for h in xrange(0, 256) :
        b[h] = h;

    for h in xrange(0, 256) :
        f = ((f + b[h]) + self.__charCodeAt(a, h % len(a))) % 256;
        i = b[h]
        b[h] = b[f]
        b[f] = i

    f = h = 0
    for q in xrange(0, len(c)) :
        h = (h + 1) % 256
        f = (f + b[h]) % 256
        i = b[h]
        b[h] = b[f]
        b[f] = i
        e = e + self.__fromCharCode(self.__charCodeAt(c, q) ^ b[(b[h] + b[f]) % 256]);

    return e

hexo+github搭建博客

技术成长第一步 搭建属于自己的技术博客

配置环境

安装Node（Node.js是建立在chrome V8上运行的js，node.js 使用事件驱动，非阻塞I/O模型使得它重量轻，效率高）
用于生成静态页面

Node.js官网下载相应平台的最新版本，一路安装即可。

安装git客户端（xcode自带Git）

申请Github（用于做博客的远程创库，域名，服务器之类的）

正式安装hexo

Node和Git都安装好后，可执行如下命令安装hexo：

npm install -g hexo

新建一个hexo目录，cd到hexo目录下，然后初始化

hexo init

hexo generate (或者 hexo -g)

启动本地服务
hexo server
浏览器输入http://localhost:4000
如果页面出不来有可能是端口被占用了
hexo server -g 端口号

目前为止，只能在自己的电脑上运行这个博客，想要不管哪里都可以看的话，要把它部署到github上

配置github

建立Repository

建立与你用户名对应的仓库，仓库名必须为[username.github.io]
eg.github注册的用户名为abc，那么仓库名必须为abc.github.io

配置hexo文件夹的_config.yml文件
vi _config.yml
翻到最下面，改成：
deploy: type: git repository: https://github.com/zenithward/zenithward.github.io.git branch: master

注意冒号后面要有一个空格

然后执行命令
hexo deploy

这时候可能会出现一个问题：ERROR Deployer not found：git
解决办法：运行 npm install hexo-deployer-git –save（如果下载失败建议重装node）

hexo g

hexo d

部署步骤

每次部署可以按以下三部来进行
hexo clean

hexo generate

hexo deploy
